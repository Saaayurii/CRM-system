# Dashboard Service (`port 3023`)

## Назначение
Управляет виджетами дашборда для каждого пользователя: персональная конфигурация набора, расположения и настроек виджетов. Каждый пользователь имеет собственный набор виджетов; данные для виджетов агрегируются на фронтенде из других сервисов.

## Ключевые эндпоинты
- `GET    /dashboard-widgets` — виджеты текущего пользователя
- `GET    /dashboard-widgets/:id` — виджет по ID
- `POST   /dashboard-widgets` — создать виджет
- `PUT    /dashboard-widgets/:id` — обновить виджет (позиция, размер, настройки)
- `DELETE /dashboard-widgets/:id` — удалить виджет

## Модели данных (Prisma)
- `DashboardWidget` — виджет (userId, type, title, config: JSONB, positionX, positionY, width, height)

## Переменные окружения
- `DATABASE_URL` — строка подключения к PostgreSQL
- `JWT_ACCESS_SECRET` — секрет для верификации JWT

## Особенности
- Изоляция по `userId`, а не по `accountId` — дашборд строго персональный
- Конфигурация виджета (какие данные показывать, фильтры) хранится в JSONB
- Сам сервис не агрегирует данные — только хранит конфигурацию; фронтенд запрашивает нужные сервисы напрямую
