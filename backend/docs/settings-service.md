# Settings Service (`port 3022`)

## Назначение
Хранит и управляет настройками приложения: системные параметры на уровне аккаунта (компании) и персональные предпочтения пользователей (язык, тема, уведомления). Системные настройки едины для всего аккаунта, пользовательские — у каждого свои.

## Ключевые эндпоинты
- `GET /system-settings` — системные настройки текущего аккаунта
- `PUT /system-settings` — обновить системные настройки аккаунта
- `GET /user-preferences` — персональные настройки текущего пользователя
- `PUT /user-preferences` — обновить персональные настройки

## Модели данных (Prisma)
- `SystemSettings` — настройки аккаунта (accountId, timezone, language, companyName, logoUrl, settings: JSONB)
- `UserPreferences` — настройки пользователя (userId, theme, language, notificationSettings: JSONB)

## Переменные окружения
- `DATABASE_URL` — строка подключения к PostgreSQL
- `JWT_ACCESS_SECRET` — секрет для верификации JWT

## Особенности
- Системные настройки привязаны к `accountId`, пользовательские — к `userId`
- Гибкие настройки хранятся в JSONB-полях для расширяемости без миграций
