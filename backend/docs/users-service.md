# Users Service (`port 3002`)

## Назначение
Управляет пользователями внутри аккаунта (компании): создание, редактирование, удаление (мягкое), смена пароля, фильтрация по роли. Все операции изолированы по `accountId`, который берётся из JWT. Также хранит данные о командах и доступности сотрудников.

## Ключевые эндпоинты
- `GET    /users` — список пользователей аккаунта (с пагинацией)
- `GET    /users/by-role/:roleId` — пользователи по роли
- `GET    /users/:id` — карточка пользователя
- `POST   /users` — создание пользователя
- `PUT    /users/:id` — обновление данных пользователя
- `PUT    /users/:id/password` — смена пароля
- `DELETE /users/:id` — мягкое удаление

## Модели данных (Prisma)
- `User` — пользователь (firstName, lastName, email, phone, roleId, accountId, avatarUrl, deletedAt)
- `Role` — справочник ролей (14 предустановленных ролей)
- `Account` — аккаунт/компания

## Переменные окружения
- `DATABASE_URL` — строка подключения к PostgreSQL
- `JWT_ACCESS_SECRET` — секрет для верификации JWT (декодирование `CurrentUser`)

## Особенности
- Email уникален в рамках одного `accountId` (не глобально)
- Мягкое удаление: поле `deletedAt` проставляется, запись не удаляется физически
- Пагинация по умолчанию: page=1, limit=20
